Manual de Taller Honda Digital

Digitalización del manual de servicio oficial Honda ESM (Accord 1998) transformando un software obsoleto exclusivo para Windows en una Progressive Web App moderna, responsive y accesible desde cualquier dispositivo

HTML5 CSS3 JavaScript Adobe Creative Cloud Responsive Mobile-First

Proyecto personal

Este es un proyecto personal actualmente en desarrollo, nacido de mi pasión por los automóviles. Soy propietario de un Honda Accord Type R CH1 del año 2000, un modelo exclusivo fabricado únicamente en Swindon (Reino Unido) para el mercado europeo con una producción muy limitada. Se trata de un vehículo que me gusta mantener en estado 100% original, sin ninguna modificación, y esta filosofía de preservación es la que me llevó a querer rescatar su documentación técnica oficial de la obsolescencia digital.

Contexto

El problema

El manual de servicio electrónico oficial de Honda (Honda ESM v1.4.0) fue distribuido en formato de software exclusivo para Windows, utilizando archivos propietarios .CAB que no son extraíbles con herramientas convencionales. Con el paso de los años, este software ha quedado obsoleto e incompatible con los sistemas operativos actuales, haciendo que un recurso técnico imprescindible para el mantenimiento del vehículo sea prácticamente inaccesible. No existe una versión digital alternativa que ofrezca este contenido de forma moderna y accesible.

Los objetivos

Rescatar todo el contenido técnico del Honda ESM original (más de 100 diagramas e imágenes técnicas que incluyen diagramas de motor, esquemas eléctricos y planos de carrocería) y convertirlo en una aplicación web progresiva (PWA) que sea instalable en cualquier dispositivo, funcione offline y ofrezca una experiencia de navegación moderna y responsive. El objetivo final es tener el manual de taller siempre accesible desde el móvil, incluso en el propio garaje sin conexión a internet.

La audiencia

Principalmente uso personal para el mantenimiento de mi propio vehículo, pero también potencialmente útil para la comunidad de propietarios de Honda Accord Type R, un grupo reducido pero apasionado de entusiastas que valoran la originalidad y el mantenimiento correcto de estos vehículos cada vez más escasos.

Enfoque

Investigación realizada

El punto de partida fue el software original Honda ESM v1.4.0, un programa exclusivo para Windows que almacena todo su contenido en archivos propietarios imposibles de extraer con herramientas convencionales. Esto significó que la única vía para rescatar el contenido fue un proceso completamente manual: capturar una a una las pantallas del software original, catalogar cada imagen, documentar cada sección y mapear toda la estructura de navegación. Se analizaron las carpetas internas del programa, identificando y extrayendo hasta el momento más de 100 imágenes técnicas entre diagramas de motor, esquemas eléctricos y planos de carrocería, aunque el manual original contiene cientos más que se irán incorporando progresivamente.

Decisiones de diseño

El reto principal fue recrear de forma fiel una arquitectura de información muy compleja, construida íntegramente de forma manual. El manual original se estructura en 16 secciones principales (desde información general, especificaciones y mantenimiento hasta motor, frenos, suspensión, carrocería, sistema eléctrico y diagramas de cableado), que a su vez se desglosan en más de 200 subsecciones con sus respectivas páginas de detalle. Toda esta jerarquía se mapeó y organizó a mano para replicar la experiencia de navegación del software original. Visualmente, se adoptaron los colores corporativos de Honda (rojo y negro) para mantener la identidad de marca, y se diseñó una interfaz con tarjetas de navegación, breadcrumbs y un sistema de búsqueda que permite localizar rápidamente cualquier procedimiento dentro de esta estructura tan extensa.

Desafíos técnicos

El mayor desafío fue la escala del trabajo manual: solo en la fase actual se han catalogado, capturado y organizado más de 100 imágenes técnicas vinculadas a más de 200 subsecciones, y el manual original alberga cientos de imágenes adicionales que aún quedan por digitalizar. Cualquier error en la asignación de una imagen a su sección correspondiente podría llevar a un mecánico a consultar un procedimiento incorrecto, por lo que el proceso de verificación fue exhaustivo. Además, conseguir que una aplicación con este volumen de contenido (que seguirá creciendo) funcione de forma fluida en un teléfono móvil, incluso sin conexión a internet dentro de un garaje, requirió optimizar cuidadosamente la carga de imágenes y la navegación entre secciones. Este proyecto sigue en desarrollo activo, ampliando progresivamente las secciones con contenido completo.

Stack técnico

HTML5 / CSS3

Lenguajes base para construir la estructura y el aspecto visual de la aplicación, permitiendo que se adapte a cualquier tamaño de pantalla y que los diagramas técnicos se lean correctamente tanto en un móvil como en un ordenador.

JavaScript

Responsable de toda la interactividad de la aplicación: la navegación entre secciones, el sistema de búsqueda, la carga progresiva de imágenes y la posibilidad de usar el manual sin conexión a internet.

Adobe Creative Cloud

Herramientas como Photoshop se han utilizado para el tratamiento y optimización de las imágenes del manual, reduciendo su peso para que la aplicación cargue rápidamente en dispositivos móviles sin perder la calidad necesaria para leer correctamente los diagramas técnicos.

Responsive Design

Diseño adaptable que permite consultar los diagramas de motor, esquemas eléctricos y planos de carrocería tanto desde el móvil en el propio garaje como en una pantalla grande para estudiar procedimientos más complejos con mayor detalle.
