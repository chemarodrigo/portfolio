Rick & Morty Character Dashboard

Personal experimentation project with Vue.js, Nuxt and Pinia: an interactive dashboard that consumes the public Rick and Morty API to browse, filter and save characters from the show's universe

Vue.js Nuxt Pinia Tailwind CSS API

Personal project

This project was born as a personal learning and experimentation exercise with Vue.js ecosystem technologies. I wanted to step out of my comfort zone and get hands-on experience with frameworks and tools widely used in professional frontend development. The public Rick and Morty API was the perfect excuse: a rich, well-documented dataset that was fun enough to keep me motivated throughout the entire development process.

Context

The problem

My frontend development knowledge was centered around vanilla JavaScript, HTML and CSS. While this foundation has allowed me to build functional projects like data visualizations and web applications, I wanted to make the leap to modern frameworks like Vue.js to understand how a real frontend application is structured: state management, reactive components, routing, authentication middleware and REST API consumption. The best way to learn a technology is to build something with it.

The objectives

Build a complete web application with Vue.js, Nuxt and Pinia that includes real digital product features: user authentication, view navigation, external API consumption, filtering and search system, favorites management and responsive design. All deployed to production to share and demonstrate that it works in a real environment.

The audience

The project is intended as a portfolio piece and learning exercise. It serves as a functional demonstration of technical capabilities with modern frameworks, aimed at showing that I can work with the tools used in current frontend development teams.

Approach

Research conducted

Before writing a single line of code, I spent time studying the official documentation of Vue.js, Nuxt and Pinia to understand each tool's philosophy and how they fit together. I analyzed the public Rick and Morty API to understand its endpoints, paginated response structure and available filtering options (by name, status, species and gender). I also researched common patterns in dashboards and data browsing applications to define a clear and functional interface.

Design decisions

The interface was designed with a dark aesthetic that matches the show's theme, using Tailwind CSS to maintain agile and consistent development. A card system was implemented to display characters with their image, name, status (alive, dead, unknown) and last known location. Navigation was split into two main views: the general character catalog with search and filters, and a favorites section where users can save their preferred characters. A simulated authentication system was added to practice managing protected routes and middleware in Nuxt.

Technical challenges

The main challenge was learning and applying three new technologies simultaneously (Vue.js, Nuxt and Pinia) with no prior experience in any of them. Managing global application state with Pinia, especially keeping the favorites list synchronized across views, required deep understanding of the reactive stores pattern. Implementing infinite pagination of the character catalog while consuming the API efficiently, without overloading requests, was another significant challenge. Setting up authentication middleware to protect routes and deploying to Vercel completed the learning of a professional frontend workflow from start to finish.

Tech Stack

Vue.js / Nuxt

Progressive JavaScript framework for building user interfaces, powered by Nuxt to manage routing, rendering and project structure in an organized and scalable way.

Pinia

Vue.js official state manager, used to centralize application data: characters loaded from the API, active filters, favorites list and user authentication state.

Tailwind CSS

CSS utility framework that enabled building a responsive and visually consistent interface quickly, with a dark aesthetic adapted to the show's theme.

Rick and Morty API

Free public REST API with data on over 800 characters. Consumed to retrieve paginated information, apply filters by name, status and species, and display each character's details.
